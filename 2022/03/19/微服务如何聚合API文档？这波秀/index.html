为什么需要聚合？

微服务模块众多，如果不聚合文档，则访问每个服务的API文档都需要单独访问一个Swagger UI界面，这么做客户端能否接受？

反正作为强迫症的我是接受不了…….

既然使用了微服务，就应该有统一的API文档入口。

如何聚合？

统一的文档入口显然应该聚合到网关中，通过网关的入口统一映射到各个模块。

本文采用Spring Cloud Gateway 聚合 Swagger 的 方式 生成API文档。

这里的单个服务不包括网关，网关需要单独配置。
